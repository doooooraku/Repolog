## 1. プロダクト概要
### 1.1 一言で
Repolog は、現場の写真を“証拠として整理し”、コメントとメタ情報（作成日時/場所/天気など）を添えて、**提出に強いA4/Letter PDFレポート**を最短で作るアプリ。

### 1.2 コア価値（不変条件）
- **証拠性**: 写真は欠けない（切り抜かない）。縦横比は維持し、枠内に縮小（contain）して載せる。
- **提出に強い**: 1ページ目（表紙）が「誰が見ても要点が分かる」構成である。
- **迷子にさせない**: 保存は “端末のどこかに勝手に保存” しない。必ずOSの保存先選択UI（Files/SAF 等）へ渡す。
- **ローカル優先**: 基本はオフラインで成立する（撮影・並べ替え・プレビュー）。外部連携は任意。
- **世界対応**: 19言語で文字化けしない・UIが崩れない。
- **端末ローカル時間**: 日付/時刻・日付キーは端末ローカル基準（海外でも自然）。

#### 対応言語（固定・19言語）
en（英語） / ja（日本語） / fr（フランス語） / es（スペイン語） / de（ドイツ語） / it（イタリア語） / pt（ポルトガル語） / ru（ロシア語） / zh-Hans（中国語・簡体字） / zh-Hant（中国語・繁体字） / ko（韓国語） / th（タイ語） / id（インドネシア語） / vi（ベトナム語） / hi（ヒンディー語） / tr（トルコ語） / nl（オランダ語） / pl（ポーランド語） / sv（スウェーデン語）

### 1.3 想定ユースケース（代表）
- 建設/設備/点検: 進捗報告、是正指示、完了報告
- 不動産/内装/清掃: 現況・施工前後の記録
- 個人: DIY、旅行、育児など「手帳のように見返す」記録

### 1.4 スコープ（今回 / 将来）
#### 今回（v1: MVP）
- レポート作成（レポート名、コメント、天気選択、位置情報ON/OFF）
- 写真追加（カメラ撮影、端末アルバムから追加）
- 写真の並べ替え / 削除 / 追加
- タイムライン（Homeで“手帳みたいに”見返す）
- PDFプレビュー（無制限）
- PDF出力（Free/Pro の制限あり）
- バックアップ（manifest.json + photos/）
- 多言語（最低: UIとPDF表記の翻訳文字列）

## 2. 用語（初心者でも迷子にならない）
- レポート（Report）: 1回の提出単位。写真＋コメント＋メタ情報を持つ。
- 写真（Photo）: レポートに紐づく画像。順序が重要。
- 表紙（Cover）: PDF 1ページ目。提出・確認の”入り口”。
- コメントページ（Comment Pages）: コメントのみのページ。長い場合は複数ページになってよい。
- 写真ページ（Photo Pages）: 写真だけ載せるページ（＋ページ番号、＋写真番号の極小ラベル）。
- タグ（Tag）: レポートに付与する文字ラベル。検索・フィルタに使う。
- ピン留め（Pinned）: 重要レポートをHome上部に固定表示する機能。
- Free/Pro: 課金プラン。制限ポイントは constraints + ADR + テストで “正” を固定する。

## 4. 画面と体験フロー（ワイヤーの“骨格”）
> UIの細部はFigmaを正にする。ここは「画面の目的」と「状態遷移」だけを固定。

### 4.1 画面一覧（最小）
- Home（タイムライン）
- レポート詳細（編集・写真管理・PDF）
- PDFプレビュー（設定: 用紙サイズA4/Letter、標準/大きく）
- 課金（Paywall）
- 設定（言語、テーマ（system/light/dark）、作成者名、位置情報ON/OFF、広告のプライバシー設定、購入セクション、バックアップ、利用規約）


### 4.2 Home（タイムライン “手帳体験”）
#### できること
- 縦にスクロールしてレポートを時系列で見返す
- 各レポートカードは「レポートの1番上の画像 + レポート名 + 日時 」を表示
- 右上: 新規レポート作成（＋）
- 検索: レポート名/コメント/タグの部分一致/日時

#### 重要な理由（なぜHomeで見返す？）
- “提出がない日”でも、未来の自分/チームのために価値が残る（継続利用の理由になる）

### 4.3 レポート詳細（編集）
#### 入力項目
- レポート名: ユーザー入力（未入力可）
- コメント: 改行OK・絵文字OK・最大4000文字（入力自体を止める）
- 天気: アイコン選択（☀️🌧️☁️❄️など）→ PDFには翻訳済みの文字列で出す
- 位置情報: 設定がONの場合のみ取得（OFFなら権限要求しない）
  - レポート作成時の1発目は自動取得
- 住所: 位置取得後に自動入力（端末言語）。ユーザーが編集可能

#### 写真管理（ここが価値の中心）
- 追加:
  - カメラ撮影（1枚ずつ）
  - アルバムから追加（複数選択OKが理想）
- 並べ替え:
  - ↑↓ボタン
- 削除:
  - 1枚削除（Undoを一度だけ出すと事故が減る）
- 上限:
  - Pro: 写真は原則無制限（ただし “PDF事故” 対策で画質設計＋警告を入れる）
  - 50枚超え: PDF作成時に警告モーダル（Generate / 戻る）

### 4.4 PDFプレビュー（無制限）
- 用紙サイズ:
  - A4 / Letter（タブ切り替え）
  - 初期値はA4
  - 選択はプレビューと出力に反映される
- レイアウト設定:
  - 標準（1ページ2枚）
  - 大きく（1ページ1枚） ※Freeでも選択・プレビュー可能（購買刺激）
- Free が “大きく” を選んだ状態で出力ボタン押下:
  - モーダル:
    - 文言: 「このレイアウトはPro限定です」
    - ボタンA: 標準（2枚）で出力する（親切）
    - ボタンB: Proにアップグレード（課金へ）

### 4.5 PDF出力（保存先選択）
- 出力ボタン押下 → 生成（必要なときだけローディング “PDFを作成中...”） → OSの保存先選択UIへ
- 保存完了の追加モーダルは出さない（OS側のUIで完了が理解できる）

## 5. 機能仕様（F）
> 受け入れ条件（合否）はテストで固定し、この文書は「境界」と「ルール」を持つ。

### レポート CRUD
- 作成: Homeの＋から新規
- 更新: レポート詳細で編集
- 削除: Homeでスワイプ or メニュー（確認ダイアログ必須）

### 写真追加（カメラ / アルバム）
- カメラ: 1枚撮影 → 即レポートに追加（順序末尾）
- アルバム: 複数選択 → 選んだ順 or 撮影日時順（選択肢はADRで固定）
- 画像形式: JPG/PNG/HEIC（端末依存）を想定。PDFに入れる前に “最終的にJPEGへ正規化” して容量を安定させる（画質設計で吸収）。

### 写真削除・順序変更
- 削除: 取り消し（Undo）を1回だけ出す（誤削除事故対策）
- 並べ替え: ↑↓ボタン
- 並べ替えの“正”は配列順（DBに orderIndex を持つ）

### メタ情報
#### 作成日時（必須）
- 表示形式: `YYYY-MM-DD HH:mm`（端末ローカル）
- ファイル名にも使う（後述）

#### 位置（任意）
- 設定ONの場合のみ取得（権限要求）
- **レポート作成時の1発目で自動取得**し、その後はユーザー操作で再取得
- 表記: 10進数、**小数点以下5桁に丸め**
  - 例: `35.68950, 139.69170`
- 取得不能・精度不十分・権限OFFのときは **「-」** を表示（空欄にしない）

#### 住所（任意 / 位置連動）
- Lat/Lng 取得後、**逆ジオコーディング**で初期入力（端末言語）
- 住所欄は **ユーザーが編集可能**（自動入力は“下書き”扱い）
- 取得不能・権限OFFのときは **「-」**

#### 天気（任意）
- UIはアイコン、PDFは翻訳済み文字列（例: ☀️→「晴れ」）

### コメント（最大4000文字）
- 改行OK、絵文字OK
- カウンタ表示（残り文字数）
- カウント: `Array.from(text).length`（絵文字ズレを減らす）
- 4000超えは入力を止める（仕様が明快）

### PDF生成（A4/Letter）
#### 目的
- “提出に強い” A4/Letter縦PDFを、崩れずに生成する。

#### 用紙サイズ（プレビューと出力で一致）
- A4 / Letter を選択可能
- 初期値は A4

#### ページ構成（固定）
- Page 1: 表紙（Cover）
- Page 2..: コメントページ（コメントのみ）
  - コメントが長い場合は複数ページになって良い（割り切り）
- コメント終了の次ページから: 写真ページ
  - 写真以外の情報は載せない（ページ番号＋写真番号のみ）

#### レイアウト（写真）
- 標準: 1ページ2枚（上下2段）
- 大きく: 1ページ1枚（Proで出力可）
- 写真の表示ルール（最重要）:
  - 写真枠（箱）を先に決める
  - 画像は枠に対して:
    - 縦横比は変えない
    - 切り抜かない
    - はみ出さないよう縮小（contain）
    - 余りは余白（白でOK）

#### ページ番号 / 写真番号
- ページ番号: `3/27` のように総ページ付き
- 写真番号: `Photo 12` を極小で隅に（現場会話で指せる）

#### 透かし（Free）
- 表紙のみ
- 右下に薄く `Created by Repolog`（目立ちすぎない）

### 出力回数 / プラン制限
> 具体制限は constraints を正にし、ADRで理由を書き、テストで固定する。
- Free:
  - 1レポート 10枚まで（出力時の上限）
  - PDF出力 月5回まで
  - 広告あり
- Pro:
  - 写真無制限
  - PDF無制限
  - テンプレ/表紙/ロゴ/透かしOFF（必要に応じて）
  - 広告なし

### 50枚超え警告（Pro向けの安全装置）
- 発火条件: PDF作成ボタン押下時に `photoCount > 50`
- モーダル:
  - タイトル: 写真枚数が多いです
  - 本文: 50枚を超えています。PDF作成に数分かかる場合があります。続けますか？
  - ボタン:
    - 作成する（Generate） → 続行
    - 戻る → レポート編集へ戻る（減らす/分ける検討ができる）

### バックアップ（エクスポート/インポート）
- エクスポート内容（最小）:
  - manifest.json（バージョン、作成日時、レポート一覧、各写真の順序/ファイル名など）
  - photos/（画像）
- PDFは含めない（pdf/ は不要）
- 保存先選択UIはPDF出力と同じ仕組みを流用
- インポート時の安全:
  - “上書き” ではなく “追加” を基本（事故が減る）
  - 競合時（同じ `report.id` / `photo.id`）は既存を優先し、重複データはスキップ
  - 同じバックアップを再インポートしても重複登録しない（ID基準で冪等）
  - DB書き込みは transaction で実行し、失敗時は rollback する

### 広告（Free）
- Freeは広告あり / Proは広告なし（明確な価値）
- UMP同意（EEA想定）で `canRequestAds=true` になるまで広告は出さない
- ポリシー対応は ADR + constraints で固定（配置・頻度・禁止画面など）

## 6. データ要件（ローカル永続化）
### 6.1 保存方針（不変条件）
- 基本データは端末ローカルに保存し、ネットワーク前提にしない
- レポート/写真メタは SQLite を推奨（検索・並べ替えが安定）
- 設定はローカルストレージ（AsyncStorage 等）
- 写真実体はアプリのドキュメント/キャッシュ領域（OSに従う）

### 6.2 データモデル（概略）
- reports:
  - id, reportName, createdAt, updatedAt, weatherKey, comment
  - authorName(string | null)
  - tags(string[] — tags_json in DB)
  - pinned(boolean)
  - locationEnabledAtCreation(bool), lat, lng, latLngCapturedAt(optional)
  - address, addressSource(auto/manual), addressLocale(optional)
- photos:
  - id, reportId, orderIndex, localUri, width, height
- exports:
  - id, reportId, exportedAt, layoutMode, pageCount, photoCount, paperSize(A4/Letter), planAtExport(Free/Pro)

## 7. 非機能要件（NFR）
### 7.1 UX/性能
- “PDF作成” は待ち時間が出る可能性がある → 進捗が分かるローディング表示
- 画像圧縮/縮小で、100MB超えのPDF事故を避ける（Pro無制限でも現実的に回る）
- エラー時も状態が壊れない（データが消えない/二重化しない）

### 7.2 アクセシビリティ（最低ライン）
- タップ領域を小さくしすぎない
- 重要文言はアイコンだけにしない（ラベル併記）
- ダークモード対応（実装済み: system/light/dark 切替）

### 7.3 セキュリティ/プライバシー（最低ライン）
- 位置情報はユーザーがON/OFFできる（OFFなら権限要求しない）
- 個人情報（アカウント等）を必須にしない（MVP）
- 課金はプラットフォームの仕組みに従う

## 8. 実装メモ（Expo/React Native 前提）
> ここは“方針”だけ。コード詳細は How-to / ADR / 実装へ。

### 8.1 PDF生成（Expo Print）
- HTML/CSS でレイアウトし、`@page { size: A4; }` / `@page { size: letter; }` を生成時に切り替える。
- CSSの単位は mm を基本（読みやすくズレにくい）
- 画像は `<img style="object-fit: contain;">` で枠内に収める。

### 8.2 保存先選択（iOS/Androidの差）
- iOS: “Save to Files” は共有シート経由になることが多い（OS標準の導線）
- Android: SAF（Storage Access Framework）でユーザーに保存先を選ばせられる

### 8.3 多言語フォント
- CJK（日本語/中国語/韓国語）で文字化けしないフォント戦略を持つ
- 必要なら PDF HTML にフォントを埋め込む（@font-face）

### 8.4 位置情報・住所（Expo Location）
- 位置情報取得: `getCurrentPositionAsync`（必要に応じて `getLastKnownPositionAsync` を併用）
- 住所取得: `reverseGeocodeAsync` で Lat/Lng → 住所
- 権限: 設定ON時のみ `requestForegroundPermissionsAsync` を呼ぶ
